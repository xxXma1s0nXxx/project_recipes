ВВЕДЕНИЕ

В современном мире информационные технологии занимают ключевое место во всех сферах
жизни общества. Интернет-платформы, ориентированные на обмен пользовательским контентом,
стремительно развиваются и привлекают миллионы пользователей по всему миру. Среди таких
платформ особое место занимают кулинарные сайты и сервисы рецептов, которые позволяют
людям не только находить готовые рецепты блюд, но и делиться собственным кулинарным
опытом, общаться с единомышленниками и формировать сообщества по интересам.

В рамках учебной практики была разработана веб-платформа «Книга Рецептов» —
многофункциональная информационная система для публикации, поиска и обсуждения
кулинарных рецептов. Данный проект представляет собой полноценное веб-приложение,
реализующее весь спектр функциональных возможностей, характерных для современных
пользовательских платформ.

Актуальность разработки подобной системы обусловлена рядом факторов. Во-первых,
кулинарные платформы являются одним из наиболее популярных видов контентных сервисов:
по данным отраслевых исследований, миллионы пользователей ежедневно ищут рецепты
в интернете. Во-вторых, разработка подобного приложения охватывает широкий спектр
задач, характерных для реальной промышленной разработки: проектирование реляционной
базы данных, реализация системы аутентификации, работа с медиафайлами, организация
поиска и фильтрации данных, построение интерактивного пользовательского интерфейса.
Всё это делает данный проект ценным учебным опытом.

Целью учебной практики является получение практических навыков разработки
веб-приложений полного цикла с применением современного технологического стека,
а также закрепление теоретических знаний в области объектно-ориентированного
программирования, проектирования баз данных и архитектуры программного обеспечения.

Для достижения поставленной цели были определены следующие задачи:
— изучение фреймворка Django и принципов его работы;
— проектирование структуры базы данных и создание моделей данных;
— реализация системы регистрации и аутентификации пользователей;
— разработка функционала создания, редактирования и удаления рецептов
  с поддержкой загрузки изображений;
— реализация системы поиска рецептов по названию и составу ингредиентов;
— разработка системы избранного, позволяющей пользователям сохранять
  понравившиеся рецепты;
— реализация механизма комментирования и оценки рецептов;
— создание персонального профиля пользователя с возможностью загрузки аватара;
— настройка административной панели для управления контентом сайта;
— организация хранения кода в системе контроля версий Git.

В ходе практики были реализованы следующие ключевые функциональные возможности
системы:

Управление пользователями. Система предусматривает полный цикл работы с учётными
записями: регистрацию новых пользователей с валидацией вводимых данных, аутентификацию
по логину и паролю, а также персональный профиль с отображением статистики активности
и возможностью загрузки фотографии профиля (аватара).

Работа с рецептами. Авторизованные пользователи могут публиковать собственные рецепты,
указывая название, описание, список ингредиентов, пошаговую инструкцию приготовления,
время готовки и загружая фотографию блюда. Автор рецепта имеет возможность
редактировать и удалять свои публикации. Администраторы системы обладают расширенными
правами и могут управлять любым контентом на платформе.

Поиск и навигация. На платформе реализован поиск рецептов по ключевым словам,
который производит поиск одновременно по названию рецепта и по составу ингредиентов.
Это позволяет пользователям находить рецепты как по названию блюда, так и по наличию
конкретного продукта в составе.

Система избранного. Каждый авторизованный пользователь может добавлять понравившиеся
рецепты в персональный список избранного и в любой момент возвращаться к сохранённым
материалам.

Комментарии и оценки. Пользователи могут оставлять текстовые комментарии к рецептам
и выставлять оценку по пятибалльной шкале. Для каждого рецепта автоматически
вычисляется средний рейтинг на основе всех полученных оценок.

Для реализации проекта был использован следующий стек технологий:
— язык программирования: Python 3.12;
— веб-фреймворк: Django 4.2 (архитектурный паттерн MVT);
— система управления базой данных: SQLite;
— фронтенд: HTML5, CSS3, Bootstrap 5, Bootstrap Icons;
— библиотека обработки изображений: Pillow;
— система контроля версий: Git / GitHub.

Архитектура приложения построена в соответствии с паттерном MVT (Model — View —
Template), который является основополагающим принципом фреймворка Django. Приложение
разделено на два модуля: accounts (управление пользователями и профилями) и recipes
(основная логика работы с рецептами). База данных включает следующие сущности:
пользователь (встроенная модель Django), профиль пользователя, рецепт, избранное,
комментарий и оценка.

Практическая значимость выполненной работы состоит в приобретении комплексных
навыков разработки веб-приложений: от проектирования моделей данных и написания
серверной логики до создания адаптивного пользовательского интерфейса и организации
командной работы с использованием системы контроля версий. Полученный опыт
соответствует требованиям, предъявляемым к junior-разработчикам в области
веб-разработки на языке Python.
